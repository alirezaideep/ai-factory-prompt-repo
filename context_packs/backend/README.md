# Context Pack: Backend

## Purpose
Provides backend development conventions, patterns, and standards for all AI agents working on server-side code generation.

## Files

| File | Description | Load When |
|------|-------------|-----------|
| `api_conventions.md` | REST API design standards, naming, versioning | Any backend API work |
| `database_patterns.md` | Schema design, migrations, query optimization | Data model changes |
| `service_layer.md` | Business logic patterns, service architecture | Service implementation |
| `auth_and_security.md` | Authentication, authorization, security patterns | Auth-related work |
| `testing_strategy.md` | Test structure, mocking, coverage requirements | Writing tests |
| `error_handling.md` | Error classification, recovery, logging | Error flow implementation |
| `event_system.md` | Event-driven patterns, pub/sub, webhooks | Inter-service communication |
| `performance.md` | Caching, connection pooling, query optimization | Performance-critical code |
